# Kickoff checklist

- [ ] Request restated as a user-visible outcome
- [ ] Goals and non-goals separated
- [ ] Architecture / compatibility constraints identified
- [ ] Relevant directories, entry points, and tests mapped
- [ ] Assumptions, risks, dependencies listed
- [ ] Work packages drafted, each with observable AC
- [ ] Validation commands chosen — Claude-runnable (not just codex self-report)
- [ ] `parallel_safe` defaulted to `no`; flipped only with disjoint-scope evidence
- [ ] Pre-flight Claude-side steps listed (network installs, tooling prep)
- [ ] Definition of Done recorded
- [ ] `.agent/current-plan.md` written
- [ ] `.agent/STATUS.md` updated with phase = planning and next step
- [ ] Blocking ambiguity (only truly blocking) surfaced to user
