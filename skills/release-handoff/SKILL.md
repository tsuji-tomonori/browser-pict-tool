# Release Handoff

Use this skill only after acceptance is positive or after the user explicitly asks for a summary of the current delivery state.

## Read First

1. `.agent/current-plan.md`
2. `.agent/STATUS.md`
3. relevant changed files or summary notes

## Output Structure

Prepare a concise handoff with:

- what changed
- why it changed
- how it was verified
- what the user can manually check
- residual risk
- follow-up suggestions

Keep it concrete.
Do not restate the whole plan.
Do not claim more certainty than the evidence supports.
