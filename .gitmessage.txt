# コミットメッセージ テンプレート
# 一般的な慣例（Conventional Commits）に従って記述してください。
# 形式: <type>(<scope>): <要約>
# 例: feat(core): 制約付きモデルの検証エラーを追加
#
# 推奨 type: feat, fix, refactor, test, docs, chore, build, ci
#
# --------------------
# 背景:
# - なぜこの変更が必要か
#
# 変更内容:
# - 何をどう変えたか
#
# 要件:
# - どの要件・課題・受け入れ条件に対応するか
#
# テスト:
# - 実行した確認コマンドと結果
# --------------------
